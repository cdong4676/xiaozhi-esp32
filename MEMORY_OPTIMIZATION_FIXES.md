# ESP_ERR_HTTPD_TASK 错误修复总结

## 问题分析

根据终端日志，系统遇到了以下关键问题：

1. **摄像头DMA缓冲区分配失败** - 内存不足导致摄像头初始化失败
2. **HTTP服务器启动失败** - `ESP_ERR_HTTPD_TASK` 错误，表示任务创建失败
3. **内存严重不足** - 可用内存仅约15KB，远低于系统正常运行需求
4. **格式化字符串错误** - `%zu` 格式不兼容问题

## 实施的修复措施

### 1. Web服务器配置优化 (`elegoo_web_server.cc`)

- **增加栈大小**: 从8192字节增加到12288字节
- **降低任务优先级**: 设置为4，避免与其他任务冲突
- **减少资源使用**: 
  - URI处理器从10减少到8
  - 响应头从8减少到6
  - 最大连接数限制为4
  - 待处理连接数限制为2
- **添加重试机制**: 失败后等待2秒重试一次
- **增强错误诊断**: 详细的错误类型分析和解决建议

### 2. 内存管理优化

#### 配置常量 (`config.h`)
```c
#define WEB_SERVER_STACK_SIZE           12288               // Web服务器栈大小
#define WEB_SERVER_INIT_RETRY_DELAY_MS  2000                // 重试延迟
#define MIN_FREE_HEAP_SIZE              (50 * 1024)         // 最小可用堆内存
#define CAMERA_MIN_FREE_HEAP_SIZE       (80 * 1024)         // 摄像头所需最小内存
#define FACE_DETECTION_MIN_HEAP_SIZE    (100 * 1024)        // 人脸检测所需最小内存
```

#### 智能内存检查
- 启动前检查可用内存
- 根据内存情况决定是否启用摄像头
- 内存不足时跳过摄像头初始化
- 条件性注册API端点

### 3. 摄像头配置优化 (`camera_image_manager.cc`)

- **降低JPEG质量**: 从12提高到15（数值越高质量越低，内存使用越少）
- **保持单帧缓冲**: 确保`fb_count = 1`以减少内存使用
- **智能初始化**: 只在内存充足时初始化摄像头

### 4. 初始化时序优化 (`elegoo_robotcar.cc`)

- **增加延迟**: WiFi连接后等待8秒再启动Web服务器（原5秒）
- **降低任务优先级**: 避免与系统关键任务冲突
- **添加内存状态日志**: 启动前记录可用内存

### 5. 格式化字符串修复

- 将所有`%zu`格式替换为`%u`
- 添加`(unsigned int)`类型转换确保兼容性
- 修复了多个文件中的格式化问题

### 6. 条件性功能启用

- **摄像头功能**: 只在内存充足时启用
- **API端点**: 根据摄像头可用性条件注册
- **人脸检测**: 根据内存情况智能启用/禁用
